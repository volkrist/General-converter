# Матрица конвертации

## PDF (MVP)

| Направление | Поведение |
|-------------|-----------|
| **image → PDF** | Одна картинка = **одна страница** (`pdf`). Юнит-тест: синтетический PNG → PDF. |
| **PDF → image** | Только **первая страница** (`pdf_render`), далее обычный encode в JPG/PNG/… Ограничение по размеру рендера (макс. сторона 2048 px) для памяти. |

Многостраничный PDF целиком **не** экспортируется.

## Что гарантируется автотестами

### 1. Юнит-тесты (`flutter test test/converter/image_converter_matrix_basic_test.dart`)

Полная матрица **5 × 5** для форматов, которые кодируются/декодируются пакетом **`image`** без нативных плагинов:

| Вход ↓ / Выход → | JPG | PNG | GIF | BMP | TIFF |
|------------------|-----|-----|-----|-----|------|
| JPG              | ✓   | ✓   | ✓   | ✓   | ✓    |
| PNG              | ✓   | ✓   | ✓   | ✓   | ✓    |
| GIF              | ✓   | ✓   | ✓   | ✓   | ✓    |
| BMP              | ✓   | ✓   | ✓   | ✓   | ✓    |
| TIFF             | ✓   | ✓   | ✓   | ✓   | ✓    |

Плюс: неизвестное расширение → ошибка.

### 2. Интеграционные тесты (только Android)

```bash
flutter test integration_test/converter_native_matrix_test.dart -d <deviceId>
```

Те же **5 синтетических входов** × **все 8 целевых форматов** (добавляются **WebP, HEIC, AVIF**).

Если какая‑то пара падает, типичные причины:

- **HEIC (выход):** нет HEIF writer на устройстве, API &lt; 28 (у нас minSdk 28).
- **WebP (выход):** ограничения `flutter_image_compress` на конкретной сборке.
- **AVIF:** зависимость от `flutter_avif` / libavif на устройстве.

## Что автотесты **не** закрывают полностью

| Вход      | Почему |
|-----------|--------|
| **WebP**  | В пакете `image` нет **encoder** WebP — синтетический файл в тесте не собрать без фикстуры. |
| **HEIC**  | Нужен реальный HEIC или нативный конвертер (`heif_converter`) — не гоняется в юнит-тесте на десктопе. |
| **AVIF**  | Вход через `decodeAvif` + `dart:ui` — в обычном `flutter test` на хосте не то же самое, что на телефоне. |

Итог: **«все во все» в смысле 8×8 файлов** на CI без Android и без фикстур **невозможно честно**. Сейчас:

- **25 направлений** базовых форматов — **проверены** каждый прогон.
- **40 направлений** (5 входов × 8 выходов) — на **Android** через integration test.
- Оставшиеся входы (webp/heic/avif как источник) — **ручная** проверка на устройстве или отдельные фикстуры в `test/fixtures/`.

## Папки

Выходной файл всегда кладётся **рядом с входным** (та же директория), имя: `basename_<timestamp>.<ext>`. Смена «папки» в UI не меняет логику — важен каталог выбранного файла.
